# 数据口径说明

## 股票池口径

系统只统计 `data/universe/a_stock_universe.csv` 中的股票。字段自动识别代码、名称、行业；如存在 `if_out`，默认 `if_out=0` 纳入，`if_out=1` 剔除。

## 行业分类口径

行业统一来自股票池文件，优先识别 `industry`、`industry_name`、`sw_industry`、`ths_industry`、`industry_L2_name`、`industry_L1_name` 等字段，不使用 Tushare 行业分类作为主依据。

## 日 K 数据口径

使用 Tushare `daily`：`ts_code, trade_date, open, high, low, close, pre_close, change, pct_chg, vol, amount`。

## moneyflow 数据口径

使用 Tushare `moneyflow`，计算小单、中单、大单、超大单与主力净流。

## 上涨占比计算

`up_ratio = pct_chg > 0 的股票数 / 有效日K股票数`。

## 涨跌幅中位数计算

对有效股票的 `pct_chg` 直接取中位数。Tushare `pct_chg` 单位为百分比，3 表示 3%。

## 涨超 3% / 跌超 3% 计算

- `gt_3_ratio = pct_chg > 3 的股票数 / 有效股票数`
- `lt_minus_3_ratio = pct_chg < -3 的股票数 / 有效股票数`

## 涨停 / 跌停计算

优先使用 Tushare `stk_limit` 的 `up_limit/down_limit`，允许极小浮点误差。若接口缺失，降级用 `pct_chg >= 9.8` 与 `pct_chg <= -9.8` 估算，并标记 `limit_calc_method=pct_chg_estimated`。

## 主力净流计算

- `elg_net_amount = buy_elg_amount - sell_elg_amount`
- `lg_net_amount = buy_lg_amount - sell_lg_amount`
- `main_net_amount = elg_net_amount + lg_net_amount`

## 行业聚合计算

按股票池行业分组，输出上涨占比、中位涨跌幅、涨停数、主力净流、超大单净流、主力净流为正占比等。

## 市场状态判断规则

规则集中在 `backend/app/calculators/rules.py`：强势、中性偏强、中性、弱势、极端弱势、结构性分化，便于后续调整。

## 涨停情绪温度口径

新增情绪模块接口：`GET /api/limitup/review?end=YYYYMMDD&days=60`。

涨停天梯与情绪温度不使用截图样本，不照搬原站明细，统一由本系统基于 Tushare `daily` + `stk_limit` + `a_stock_universe.csv` 实际计算。连板天梯、最高连板、情绪温度中的连板奖励均剔除 ST / *ST / SST / 退市整理股；ST 判断优先根据股票池中的股票名称自行识别。

涨停/跌停判断：

```text
limit_up = close >= up_limit - 1e-4
limit_down = close <= down_limit + 1e-4
```

如果 `stk_limit` 缺失，则降级为：

```text
limit_up = pct_chg >= 9.8
limit_down = pct_chg <= -9.8
```

连板数计算：按交易日顺序逐日累加，某股票当日涨停且上一交易日也涨停，则 `streak = previous_streak + 1`；否则从 1 开始。

情绪值口径：

```text
emotion = 涨停数量 - 跌停数量 + 三板以上数量×2 + max(最高连板-2, 0)×3
```

Selected Day 明细包括：涨停数量、跌停数量、情绪值、三板以上数量、最高连板、最高标。

注意：Tushare 日K与 `stk_limit` 不包含封单金额、首次封板时间、炸板次数等盘口字段，因此前端不展示这些字段；如未来接入东方财富/同花顺盘口源，需要单独标注数据来源。
