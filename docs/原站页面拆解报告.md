## 页面结构

原站为单页结构：

1. 顶部大标题区：`Limit-up Review / 涨停板复盘`。
2. 顶部页签：`连板复盘`、`强趋势复盘`。
3. 控制区：结束日期、天数、刷新按钮。
4. 状态条：数据源、更新时间、最新情绪。
5. 图表区：每日情绪温度。
6. 天梯区：涨停天梯，横向按日期展示。
7. 明细区：选中日期明细。
8. 强趋势视图包含趋势摘要、行业图表、K线图、趋势表格与明细。

## 数据展示字段

原站接口 `/api/review?days=20&end=YYYYMMDD` 返回字段包括：日期、涨停数量、跌停数量、情绪分、候选池、股票代码、名称、行业、连板数、封单金额、炸板次数、角色标签等。

本次系统需求不同，改造为 A 股每日复盘，核心字段调整为：市场状态、价格宽度、资金流、行业共振、数据覆盖率、自动复盘文本。

## 图表类型

- 本次复现：保留原站纸张底色、黑色边框、顶部状态条、卡片和表格顺序；图表采用 Recharts 展示市场宽度柱状图与行业共振趋势图。

## 本次复现和改造方案

- 不重做完全不同 UI，沿用原站的复古纸张背景、黑色描边、大标题、状态条、panel、表格等视觉语言。
- 由于目标从“涨停板复盘”升级为“A股每日复盘系统”，保留布局精神与交互方式，替换为市场宽度、moneyflow、行业共振等数据模块。
- 后端采用 FastAPI + pandas + Tushare + 本地 Parquet/CSV 缓存 + SQLite 索引。
- 股票池和行业分类统一来自 `data/universe/a_stock_universe.csv`。
